{platform_name_cn}为云使用者提供一种随时获取的、弹性的计算能力，这种计算能力，我们称之为云服务器。

云服务器就是一台配置好的服务器，它有使用者期望的硬件配置、操作系统和网络配置。{platform_name_cn}提供的云服务器类型包含 `基础型` 和 `企业型` 两种。

* 基础型云服务器
+
依赖的物理资源池是不固定的，包括 CPU、硬盘等配置都不固定。所以，当创建的基础型云服务器的安置策略，在安置到基础型资源池后，其所落在的物理机配置也不固定。
+
基础型云服务器提供从 1C 1G - 8C 32G 的可选配置，是面向个人或中小团队用户推出的入门型云服务器类型。它以极高的性价比满足低负载应用、轻量 Web 应用、开发测试环境、轻量级企业服务等使用场景需求。
+

include::../../../../../_components/note.adoc[]

* IOPS 范围参考：500 -- 2500
* 单盘 IOPS 计算公式： min{500 + 8 * 容量, 2500}
* IO 吞吐范围参考：36 MB/s - 100MB/s
* 单盘 IO 吞吐计算公式：min{36 + 0.15 * 容量, 100 } MBps

include::../../../../../_components/table_footer.adoc[]

* 企业型云服务器
+
适用于企业级客户。企业级客户如果对资源的性能有明确的需求，可以选用企业型云服务器，从而获得可靠的计算实例。
+
企业型云服务器提供从 2C 4G - 32C 512G 的可选配置，企业型 e2 云服务器采用独享 CPU 模式，可提供更高更稳定的计算性能，并针对人工智能工作负载进行优化，满足企业更严苛的性能要求。
+

include::../../../../../_components/note.adoc[]

* IOPS 范围参考：2000 -- 30000
* 单盘 IOPS 计算公式： min{2000 + 30 * 容量, 30000}
* IO 吞吐范围参考：128MB/s - 320 MB/s
* 单盘 IO 吞吐计算公式：min{128 + 0.5 * 容量, 320 } MBps

include::../../../../../_components/table_footer.adoc[]



== 功能描述

云服务器资源列表拥有监控模式和普通列表模式两种，显示当前{platform_name_cn}中每台云服务器的状态、资源 ID、云服务器类型、云服务器来源（分为系统、租户两种）、所属区域、所属计算节点、镜像名称/ID、云服务器配置、所属网络名称与 ID、网卡类型、云服务器拥有者邮箱/ID、创建时间等信息。

同时，该页面还支持资源迁移、转移租户、重置 fstab 等操作。

+
image::/images/boss/manual/resource_mgt/cloud_server_2.png[]

. 点击 *资源监控*，可以获得指定时间段内的 CPU 使用率、内存使用率、系统盘吞吐量、硬盘每秒读写次数、系统盘使用率、网络包转发率时序监控数据。
+
[cols="2,8", options="header"]
|===
|参数
|参数说明

|CPU 占用率
|这一指标用于衡量当前主机上运行的所有程序对 CPU 计算性能的利用情况，反映主机的繁忙状态。

|内存占用率
|内存一般被认为是 CPU 寄存器的高速缓存，该项指标用于衡量主机内存利用情况，这项指标配合 CPU 利用率共同反映主机运行状态。

|吞吐量
|每秒的 I/O 吞吐量，一般受 IOPS 的影响。

|IOPS （Input / Output Per Second）
|即每秒的输入输出量（或读写次数），是衡量磁盘性能的主要指标之一。

|网络（Kbps）
|每秒传送多少千字节，反映当前主机在网络上的传输速率。

|包转发率（pps）
|反映当前主机网络吞吐量的指标。
|===

. 点击**监控日志**，可以浏览云服务器的操作日志，每项关于云服务器的任务信息与状态都按时间顺序显示，以支持使用者操作回溯。

include::../../../../../_components/note.adoc[]

* 云服务器状态为 *已删除* 时，不可执行迁移云服务器操作。

. （仅在选择了手动选择计算节点模式时执行该操作）选择节点。

== 转移租户

将租户下的云服务器资源转移给其他租户。云服务器状态为**运行中**或**已关机**时，才能执行转移租户操作。

include::../../../../../_components/note.adoc[]

* 迁移的资源不可以改变 Zone。
* 系统资源的云服务器，不支持租户转移。
* 进行租户转移时，系统不会对接收方的账户余额做校验。如果资源转移中，因为余额不足导致部分资源转移失败，系统会将失败的资源 `owner` 回滚到原租户。

include::../../../../../_components/table_footer.adoc[]




详细操作步骤说明如下：

. 进入link:#_查看云服务器列表[云服务器列表]页面。在搜索框内输入目标云服务器的名称、ID 或者 IP 搜索目标云服务器。

. 点击目标云服务器的更多操作中的**转移租户**进行单个资源的转移。
+
image::/images/boss/manual/resource_mgt/vm_option_more.png[vm_option_more,100%]


. 弹出转移云服务器对话框，用户根据页面提示信息，选择填写相关参数后，点击**转移**。
+
image::/images/boss/manual/resource_mgt/vm_owner_change.png[vm_owner_change,60%]
+

include::../../../../../_components/note.adoc[]

* 资源租户转移过程会在系统中生成一个 Job，这个 Job 在link:../../../om_tools/operation_log/[操作日志]中可以查看到详细结果。
* 若用户勾选了 `任务完成时通知我`，则系统会将执行结果同步给管理员。
* 只可针对租户级别的云服务器进行租户转移。
* 用户能选择硬盘，公网 IP 以及备份等资源与云服务器一起转移。

include::../../../../../_components/table_footer.adoc[]

. BOSS 支持批量操作云服务器进行租户转移。用户在云服务器列表页面，勾选多个目标云服务器后，点击**转移租户**。
+
image::/images/boss/manual/resource_mgt/vm_owner_change_more.png[vm_owner_change_more,100%]

. 弹出**批量转移租户**对话框，根据页面提示信息，依次确认需要转移的云服务器与云服务关联资源后，并设置转移参数后，点击**转移**。



== 重置 fstab

fstab（file system table）文件系统表。用于存放文件系统信息，应用程序读取这个文件，可根据其内容完成自动挂载的工作。

include::../../../../../_components/note.adoc[]
* 云服务器状态为 *已关机* 时，可执行重置 fstab 操作。

* 重置 fstab 将会把当前的 fstab 文件恢复到创建虚机时到状态，以解决因为错误配置 fstab 文件导致的无法开机的问题。

* 云服务器重启后，请正确配置 fstab 文件，修改完 fstab 请使用 `mount -a` 检查 fstab 文件是否正确。

include::../../../../../_components/table_footer.adoc[]

. 在 *云服务器* 页面搜索框搜索待操作云服务器的名称、ID 或者 IP，按 *Enter*，找到待操作的云服务器。
. 在待操作云服务器所在行，点击 image:/images/boss/manual/icon.png[]，点击 *重置 fstab*。
+
弹出 *重置 fstab* 对话框。

. 输入操作原因，点击 *重置*。

== 启用网卡多队列

网卡多队列作为一种技术手段，可用于解决网络 IO QoS （quality of service）问题。它的工作原理是将多个队列产生的中断分布到 CPU 的多个不同核上，实现负载均衡，避免单个核被占用到 100% 而其他核还处于空闲的情况，多队列网卡的每个队列的中断都是使用单独的中断线，提升网络吞吐率（PPS）和带宽性能。

include::../../../../../_components/note.adoc[]
云服务器状态为 *已关机* 时，可执行启用网卡多队列操作。

include::../../../../../_components/table_footer.adoc[]

. 在 *云服务器* 页面搜索框搜索待操作云服务器的名称、ID 或者 IP，按 *Enter*，找到待操作的云服务器。
. 在待操作云服务器所在行，点击 image:/images/boss/manual/icon.png[]，点击 *启用网卡多队列*。
+
弹出 *启用网卡多队列* 对话框。
+
image::/images/boss/manual/resource_mgt/cloud_server_4.png[]

. 输入操作原因，点击 *确认*。
