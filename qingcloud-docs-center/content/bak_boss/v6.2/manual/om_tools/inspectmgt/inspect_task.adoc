巡检管理页面展示了{platform_name_cn}上所有巡检任务列表，包括任务状态、执行状态、名称、创建人、执行类型、周期、执行时刻、最近执行时间及创建时间。用户在该页面可新增、修改、执行、删除巡检任务。




== 巡检任务列表

. 成功登录 BOSS。

. 点击**运维工具** > **巡检管理** > **巡检任务**，进入**巡检任务**列表页面。
+
image::/images/boss/manual/om_tools/inspect_task_1.png[]

== 新增巡检任务

. 进入link:#_巡检任务列表[巡检任务]列表页面，点击**新增任务**，弹出**新增巡检任务**对话框。
+
image::/images/boss/manual/om_tools/inspect_task_2.png[inspect_task_2,80%]

. **新增巡检任务**对话框各参数说明如下，用户根据需求配置完成后，点击**创建**。
+
[cols="2,8a", options="header"]
|===
|参数类型|参数说明
|名称|巡检任务的名称。
|执行类型 a|
* `仅执行一次` 表明该巡检任务仅执行一次，需同时配置 `执行时刻`。
* `重复执行` 表明该巡检任务会执行多次，需同时配置 `周期`。
|执行时刻|选择具体执行的日期和时间。当 `执行类型` 设置为 `仅执行一次` 时，展示该参数。
|周期|包括按小时重复、按日重复、按周重复、按月重复。当 `执行类型` 设置为 `重复执行` 时，展示该参数。
|通知组 a|
* 可选项。指定通知组，巡检报告生成后将会发送通知至 `通知组` 中的邮箱。
* 可在下拉列表中选择已有的通知组，也可点击 **+ 创建通知组**创建通知组。创建通知组的步骤请参考link:../../../system_management/system_setting/notification_group/#_创建通知组[前文内容]。
* 巡检报告内容可参考link:../../../system_management/system_setting/notification_group/#_巡检报告[前文内容]。
|规则选择 a| 确认巡检规则。巡检规则分为基础设施巡检、管理服务巡检以及应用巡检。38 个管理服务和组件的巡检，共 55 个巡检项，包括{platform_name_cn}的账号、告警、消息通知等服务，Memcached、pgpool，pg等组件，全部巡检项请参考link:../inspect_list[巡检项说明]。

* `基础设施巡检`：对于计算节点和管理节点的巡检。若选择该选项，需同时确定 `按区域选择` 还是 `按安置组选择`。
* `管理服务巡检`：对于支持{platform_name_cn}的各种微服务的巡检。
* `应用巡检`：对于 AppCenter 所有应用的巡检。

include::../../../../../_components/note_in_table.adoc[]

* 需要购买相关 PaaS 服务，才能使用**应用巡检**功能。
* 应用巡检需要应用的开发者定义应用版本的监控项，监控服务获得应用监控项后可对集群进行监控告警。管理员在巡检任务中勾选的应用，会对当前选中的所有应用集群进行巡检，最终出具巡检报告，巡检结果反馈的是当前集群及节点的状态。

include::../../../../../_components/footer_in_table.adoc[]

|===

. 返回巡检任务页面，可查看到新增的巡检任务显示在列。


== 查看任务详情

进入link:#_巡检任务列表[巡检任务]列表页面，点击目标巡检任务的**详情**操作。进入目标巡检任务的详情页面。该页面列表显示当前巡检任务的执行历史以及基本信息。

image::/images/boss/manual/om_tools/inspect_task_4.png[]

include::../../../../../_components/note.adoc[]

如上图所示，用户也可在该页面对巡检任务执行 `删除`、`禁用`、`修改` 以及 `立即执行` 等操作。

include::../../../../../_components/table_footer.adoc[]

== 批量管理任务

进入link:#_巡检任务列表[巡检任务]列表页面，勾选多条目标巡检任务，可进行**批量启动任务**、**批量禁用任务**、**批量删除任务** 的操作。

image::/images/boss/manual/om_tools/inspect_task_5.png[]


== 更多操作

进入link:#_巡检任务列表[巡检任务]列表页面，点击目标巡检任务的更多操作。用户在该页面可进行 `修改`、`立即执行`、`删除` 以及 `禁用` 巡检任务。

image::/images/boss/manual/om_tools/inspect_task_3.png[]

include::../../../../../_components/note.adoc[]

* 点击 `立即执行` 后，该任务会立即执行一次。若前期设置了巡检任务 `仅执行一次`，执行了 `立即执行` 后，仍然会再次执行一次。

* 针对正在执行中的巡检任务，也可以执行 `删除` 操作。

* `修改` 操作对正在执行中的巡检任务不生效。

include::../../../../../_components/table_footer.adoc[]

== 一键巡检


. 进入link:#_巡检任务列表[巡检任务]列表页面，点击**一键巡检**，弹出**一键巡检**对话框。系统推荐对所有设施、服务、资源进行一键巡检，或用户也可以自定义勾选具体巡检小类，进行一键巡检。
+
image::/images/boss/manual/om_tools/inspect_task_6.png[inspect_task_6,80%]

. 用户根据需求配置完成后，点击**立即巡检**。
+

include::../../../../../_components/note.adoc[]

一键巡检是 `立即执行` 的操作，且仅执行一次。

include::../../../../../_components/table_footer.adoc[]

== 阈值管理

BOSS 支持对巡检时的检查项阈值进行配置，使得用户可针对具体的环境，配置不同的检查条件。

. 进入link:#_巡检任务列表[巡检任务]列表页面，点击**阈值管理**，弹出**阈值管理**对话框。默认进入**基础设施**阈值设置页面。用户可根据需求，调整该页面的阈值后，点击**保存**。
+
image::/images/boss/manual/om_tools/inspect_task_7.png[inspect_task_7,60%]


. 点击**管理服务**，进入**管理服务**阈值调整页面。用户可根据需求，调整该页面的阈值后，点击**保存**。
+
image::/images/boss/manual/om_tools/inspect_task_8.png[inspect_task_8,60%]
