== 基础设施巡检

[cols="1,2,3", options="header"]
|===
|分类
|巡检项
|检查项条件

.6+|计算资源检查
|系统负载检查
|load average 不超过核心数的 1.2 倍，检查通过。

|内存使用率检查
|内存使用率不超过 70%，检查通过。

|CPU 等待率检查
|CPU WA（CPU 等待率）不超过 5%，检查通过。

|CPU 软件中断率检查
|CPU SI（CPU 软件中断率）不超过 10%，检查通过。

|交换分区使用率检查
|交换分区使用率不超过 15%，检查通过。

|CPU 使用率巡检
|CPU 使用率不超过 70%，检查通过。

.2+|进程检查
|僵尸进程（Zombie）检查
|僵尸进程不超过 50，检查通过。

|异常进程（D 进程）检查
|异常 D 进程不超过 1，检查通过。

.3+|存储资源检查
|磁盘 inode 使用率检查
|磁盘 inode 使用率不超过 80%，检查通过。

|磁盘使用率检查
|块存储使用率不超过 80%，检查通过。

|IO 占用率检查
|占用率不超过 80%，检查通过。

.3+|网络资源检查
|网络错包率检查
|错包率出入不超过 5%，检查通过。

|网络丢包率检查
|丢包率出入不超过 5%，检查通过。

|物理机网卡 bond 状态检查
|bond 状态及其端口都是 up 状态，检查通过。

.3+|操作系统检查
|系统时间检查
|NTP 服务正常运行以及系统时间差异小于 3s 时，检查通过。

|闪存卡寿命检查
|若有 916、910 型号卡， OP 空间不超过 80% 则为正常。

|supervisor 状态检查
|supervisor 服务运行正常，检查通过。
|===

== 应用巡检

include::../../../../../_components/note.adoc[]
目前仅 Etcd、MySQL Plus、OpenSearch、Redis Standalone、Redis Cluster、MongoDB 支持监控版本功能，所以仅支持对这些应用进行巡检。

include::../../../../../_components/table_footer.adoc[]

**监控项是由产品本身决定的，所以每个应用的巡检项都存在差异。**
