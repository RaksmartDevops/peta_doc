== 背景信息

{platform_name_cn}用户可以通过 IAM 身份向应用程序或他人来委派自己名下{platform_name_cn}服务和资源的访问权限。
当用户在{platform_name_cn}创建身份后，即表示该身份的指定实体与本账户之间建立了信任关系，该实体（即身份的信任载体）可通过代入此身份而获得本账户的一个临时访问凭证。用户可以通过修改身份属性来调整该凭证的会话有效时长，也可以通过调整身份上附加的策略来控制该凭证的可访问权限范围。用户创建的身份详细信息统一呈现在 *云计算运维管理平台* 的 *IAM: 客户身份* 页面，以供管理员查看。

== 场景描述

该任务指导用户进入 *IAM：客户身份* 页面，以查看{platform_name_cn}用户创建的客户身份列表和详情。

== 功能描述

*IAM: 客户身份* 页面罗列了{platform_name_cn}用户创建的身份列表及其状态、身份名称/ID、描述、信任载体类型、绑定策略、客户 ID 和更新时间等信息。

目前 IAM 服务支持三种类型的身份：账户、主机、应用

，其中账户和应用类型身份创建时需要指定信任载体（principal），主机身份则无需指定。

== 前提条件

已获取 BOSS 的登录账号和密码。

== 操作步骤

. 使用浏览器登录 BOSS。
. 选择 *资源管理* > *IAM* > *客户身份*，进入客户身份页面，可查看客户身份列表及详细信息。
+
image::/images/boss/manual/resource_mgt/user_role_operation.png[]

. 点击客户身份名称，进入客户身份详情页面。
+
在 *信任载体详情* 页签，可查看身份详情和控制台可见模块信息。
+
在 *关联策略* 页签，可查看该身份下关联的策略列表以及列表下的摘要和 JSON 信息。
+
image::/images/boss/manual/resource_mgt/user_role_view.png[]
