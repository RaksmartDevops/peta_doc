== 场景描述

该任务指导管理员进入客户消费页面，查看客户账户以及所有子账户下的消费详细信息。并根据实际需要导出消费明细。

== 功能描述

客户消费页面根据时间段展示客户总数及消费总额、客户消费额、账户明细、资源明细（汇总视图和详细视图）、项目明细。

== 前提条件

已获取 BOSS 的登录账号和密码。

== 操作步骤

. 使用浏览器登录 BOSS。
. 选择 *运营管理* > *计费管理* > *客户消费*，进入客户消费页面。
+
image::/images/boss/manual/operationmgt/user_consumption_operation.png[]

. 在 *客户消费* 页面左上角选择日期，可查看该时间段客户的消费信息。
. 点击客户名称，可查看该客户名下所有项目明细数据。
+
* 在 *账户明细* 页签，可查看改客户的所有账户列表、客户名称、账户类型、资源总数、应付金额、优惠券抵扣、实付金额以及消费占比信息。客戶消费额等于所有主账户和子账户的实付金额总和。
+
* 在 *资源明细* 页签，可查看改客户名下使用的资源、资源总数、计费方式、应付金额、优惠券抵扣、实付金额以及消费占比。客户消费额等于所有资源实付金额总和。
+

include::../../../../../_components/note.adoc[]
该页签分为 *汇总视图* 和 *详情视图* 两种呈现方式，*汇总视图* 方便管理员进行数据统计，*详情视图* 便于管理员查看数据详情。

include::../../../../../_components/table_footer.adoc[]
+
* 在 *项目明细* 页签，可查看该客户使用的项目列表、项目名称。应付金额、优惠券抵扣、实付金额以及消费占比。由于不是所有资源都会加入到项目中，所以客户消费额与项目实付金额没有直接联系。
+

include::../../../../../_components/note.adoc[]

消费占比可分为 *按应付算消费占比* 和 *按实付算消费占比* 两种，可根据需要勾选查看对应消费占比。

include::../../../../../_components/table_footer.adoc[]


=== 导出消费账单

. 在 *客户消费* 页面，点击 *导出客户消费额*，可导出客户消费列表。

. 在 *客户消费* 页面，分别选择 *账户明细*、*资源明细*、*项目明细* 页签，点击对应 *导出账户明细*、*导出资源明细* 或 *导出项目明细*，可导出对应明细列表。

. 导出后，点击 *报表管理*，在对应的模块名称所在行点击文件名称，对应的消费额列表或账单明细会保存到本地。
