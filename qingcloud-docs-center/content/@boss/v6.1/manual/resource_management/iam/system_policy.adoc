== 背景信息

预置策略是预先已定义由平台所有用户共享的策略，该策略管理员在 *云计算运维管理平台* 创建并设置，创建完成并发布后，{platform_name_cn}用户可在 *系统预置* 中的策略列表下勾选使用。

== 场景描述

该任务指导管理员进入 *IAM: 系统预置策略* 页面，并执行创建、修改、删除或发布预置策略等操作。

== 功能描述

*IAM: 系统预置策略* 该页面展示了管理员创建的所有预置策略的状态、策略名称/ID、描述、当前版本、待发布版本以及更新时间信息。

== 前提条件

已获取 BOSS 的登录账号和密码。

== 操作步骤

. 使用浏览器登录 BOSS。
. 选择 *资源管理* > *IAM* > *系统预置策略*，进入系统预置策略页面，可查看预置策略列表和详情。
+
image::/images/boss/manual/resource_mgt/system_policy_operation.png[]

. 点击预置策略名称，进入预置策略详情页面。
+
在 *权限* 页签，可查看预置策略的摘要和 JSON 信息，点击 *修改策略*，可通过修改 JSON 修改预置策略的服务和操作限制。
+
在 *版本管理* 页签，选择版本号可查看该版本的 JSON 信息；点击 *策略模拟*，选择服务和操作，可查看该策略下具体操作设置的允许或拒绝策略；若需使用历史版本的预置策略可点击 *应用*。
+
在 *历史记录* 页签，可查看该预置策略历史操作记录状态。
+
image::/images/boss/manual/resource_mgt/system_policy_view.png[]
+
=== 创建预置策略并发布

. 在 *IAM: 系统预置策略* 页面右上角点击 *创建预置策略*。
+
弹出 *创建预置策略* 对话框。

. 输入 JSON 信息，配置策略。
+
JSON 配置格式详情请参见{platform_name_cn}文档 “访问鉴权管理 > 常见问题 > JSON 编辑格式及参数说明”。

. 点击 *下一步*。
+
进入 *审阅* 页签。

. 输入策略名称和描述。
. 点击 *创建*。

=== 发布预置策略

include::../../../../../_components/note.adoc[]
预置策略状态为 *未发布* 时，可执行发布预置策略操作。

只有发布后的预置策略才可以在{platform_name_cn}显示供用户使用。

include::../../../../../_components/table_footer.adoc[]

. 在未发布策略名称所在行末尾点击image:/images/boss/manual/icon.png[]。点击 *发布*。
. 输入处理意见，点击 *确认发布*。

=== 删除预置策略

include::../../../../../_components/note.adoc[]
预置策略状态为 *未发布* 时，可执行删除预置策略操作。

include::../../../../../_components/table_footer.adoc[]

. 在未发布策略名称所在行末尾点击image:/images/boss/manual/icon.png[]，点击 *删除策略*。
+
弹出 *删除策略* 对话框。

. 输入操作原因，点击 *删除*。

=== 修改预置策略名称和描述

. 在待操作预置策略名称所在行点击image:/images/boss/manual/icon.png[]，点击 *修改信息*。
. 重新输入预置策略名称和描述。
. 点击 *修改*。
